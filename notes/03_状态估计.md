# 03. 状态估计详解

## 1. 基本原理

### 1.1 为什么需要状态估计？

**问题**: 机器人需要知道自己的基座位置和速度，但：
1. **IMU 有漂移**: 积分误差累积
2. **没有 GPS**: 室内环境无绝对位置
3. **没有外部定位**: 如动捕系统

**解决方案**: 融合多种传感器
- **IMU**: 姿态、角速度、线加速度 (高频但有漂移)
- **关节编码器**: 关节位置、速度 (精确但只有相对信息)
- **运动学**: 通过正运动学计算脚位置 (提供绝对约束)

### 1.2 估计什么？

**估计状态** (6维):
```
x_est = [p_b, v_b]
        ├─ p_b: 基座位置 (3维) [x, y, z]
        └─ v_b: 基座速度 (3维) [vx, vy, vz]
```

**已知量**:
- IMU: 基座姿态 R、角速度 ω、线加速度 a_imu
- 关节: 关节角度 q、关节速度 q̇
- 接触: 脚是否触地 (contact_flag)

## 2. 卡尔曼滤波

### 2.1 什么是卡尔曼滤波？

**卡尔曼滤波 (Kalman Filter)** 是最优状态估计器：

$$
prediction step (Predict):
  \hat{x}(k|k-1) = F * \hat{x}(k-1|k-1) + B \cdot u(k)
  \mathbf{P}(k|k-1) = F * \mathbf{P}(k-1|k-1) \cdot F^{T} + Q

update step (Update):
  K = \mathbf{P}(k|k-1) \cdot H^{T} * (H * \mathbf{P}(k|k-1) \cdot H^{T} + R)^-1
  \hat{x}(k|k) = \hat{x}(k|k-1) + K \cdot (z(k) - H * \hat{x}(k|k-1))
  \mathbf{P}(k|k) = (I - K \cdot H) * \mathbf{P}(k|k-1)

where:
  \hat{x}: state estimate
  P: covariance matrix
  K: Kalman gain
  F: state transition matrix
  H: observation matrix
  Q: process noise covariance
  R: measurement noise covariance
$$

### 2.2 线性卡尔曼滤波

**系统模型**:
```
状态转移: x(k+1) = F * x(k) + B * u(k) + w(k)
观测模型: z(k) = H * x(k) + v(k)

其中:
  w(k) ~ N(0, Q): 过程噪声
  v(k) ~ N(0, R): 观测噪声
```

**预测步** (IMU 积分):
```
状态转移矩阵:
  F = [I_3, dt*I_3]
      [0,   I_3   ]

控制输入矩阵:
  B = [0.5*dt²*I_3, 0]
      [dt*I_3,      0]

控制输入:
  u = [a_imu, 0]  (IMU 线加速度)

预测:
  p̂(k+1|k) = p̂(k|k) + v̂(k|k) * dt + 0.5 * a_imu * dt²
  v̂(k+1|k) = v̂(k|k) + a_imu * dt

协方差预测:
  P(k+1|k) = F * P(k|k) * F^T + Q
```

**更新步** (运动学校正):
```
当脚触地时:
  观测模型: z = p_foot_measured = FK(q)
  观测矩阵: H = [I_3, 0]
  
  预测观测: ẑ = p_b + R * r_foot
  
  残差: y = z - ẑ = p_foot_measured - (p_b + R * r_foot)
  
  卡尔曼增益: K = P * H^T * (H * P * H^T + R)^-1
  
  状态更新:
    p̂(k|k) = p̂(k|k-1) + K * y
    v̂(k|k) = v̂(k|k-1) + K_v * y
  
  协方差更新:
    P(k|k) = (I - K * H) * P(k|k-1)
```

### 2.3 为什么用线性卡尔曼滤波？

**优势**:
1. **简单**: 计算量小，实时性好
2. **稳定**: 收敛性好
3. **易于调参**: 只需调节 Q, R

**劣势**:
1. **线性假设**: 不能处理强非线性
2. **精度有限**: 不如扩展卡尔曼滤波 (EKF)

**为什么够用？**:
- 四足机器人运动相对平缓
- IMU 频率高 (500Hz)，积分误差小
- 运动学校正频繁 (每步都校正)

## 3. 代码分析

### 3.1 LinearKalmanFilter 类

**文件**: `legged_estimation/src/LinearKalmanFilter.cpp`

**主要功能**:
1. 预测步 (IMU 积分)
2. 更新步 (运动学校正)
3. 输出估计状态

**关键函数**:

```cpp
void LinearKalmanFilter::updateFromImu(const ros::Time& time, 
                                       const ros::Duration& period) {
    // 1. 获取 IMU 数据
    Vector3d a_imu = imuSensorHandle_.getLinearAcceleration();
    Matrix3d R = imuSensorHandle_.getOrientation();
    
    // 2. 转换到世界坐标系
    Vector3d a_world = R * a_imu - Vector3d(0, 0, 9.81);  // 减去重力
    
    // 3. 预测步
    Vector6d x = stateEstimate_;
    Vector3d p = x.head(3);
    Vector3d v = x.tail(3);
    
    double dt = period.toSec();
    p = p + v * dt + 0.5 * a_world * dt * dt;
    v = v + a_world * dt;
    
    // 4. 协方差预测
    Matrix6d F = Matrix6d::Identity();
    F.block<3, 3>(0, 3) = dt * Matrix3d::Identity();
    
    Matrix6d Q = Matrix6d::Identity();
    Q.block<3, 3>(0, 0) = dt * dt * processNoisePosition_;
    Q.block<3, 3>(3, 3) = dt * processNoiseVelocity_;
    
    covariance_ = F * covariance_ * F.transpose() + Q;
    
    // 5. 保存预测状态
    stateEstimate_.head(3) = p;
    stateEstimate_.tail(3) = v;
}

void LinearKalmanFilter::updateFromContact(const ros::Time& time) {
    // 1. 遍历所有脚
    for (size_t i = 0; i < 4; ++i) {
        // 2. 检查是否触地
        if (contactHandles_[i].isContact()) {
            // 3. 计算脚的世界位置 (运动学)
            Vector3d p_foot_world = computeFootPosition(i);
            
            // 4. 期望脚位置 (应该在地面上)
            Vector3d p_foot_expected = Vector3d(0, 0, 0);  // 假设地面 z=0
            
            // 5. 残差
            Vector3d y = p_foot_expected - p_foot_world;
            
            // 6. 卡尔曼增益
            Matrix3d H = Matrix3d::Identity();
            Matrix3d S = H * covariance_.block<3, 3>(0, 0) * H.transpose() + measurementNoise_;
            Matrix3d K = covariance_.block<3, 3>(0, 0) * H.transpose() * S.inverse();
            
            // 7. 状态更新
            stateEstimate_.head(3) += K * y;
            
            // 8. 协方差更新
            Matrix6d I_KH = Matrix6d::Identity();
            I_KH.block<3, 3>(0, 0) -= K * H;
            covariance_ = I_KH * covariance_;
        }
    }
}

Vector3d LinearKalmanFilter::computeFootPosition(size_t footIndex) {
    // 1. 获取关节角度
    Vector3d q = getLegJoints(footIndex);
    
    // 2. 正运动学
    Vector3d p_foot_base = forwardKinematics(q);
    
    // 3. 转换到世界坐标系
    Matrix3d R = imuSensorHandle_.getOrientation();
    Vector3d p_base = stateEstimate_.head(3);
    
    Vector3d p_foot_world = p_base + R * p_foot_base;
    
    return p_foot_world;
}
```

### 3.3 状态估计接口

**StateEstimateBase 类**:
```cpp
class StateEstimateBase {
public:
    virtual void updateFromImu(const ros::Time& time, const ros::Duration& period) = 0;
    virtual void updateFromContact(const ros::Time& time) = 0;
    
    Vector6d getState() const { return stateEstimate_; }
    Matrix6d getCovariance() const { return covariance_; }
    
protected:
    Vector6d stateEstimate_;      // [p_b, v_b]
    Matrix6d covariance_;          // 协方差矩阵
};
```

**FromTopicEstimate 类** (作弊模式):
```cpp
class FromTopicEstimate : public StateEstimateBase {
public:
    FromTopicEstimate(ros::NodeHandle& nh) {
        // 直接从 Gazebo 获取真实状态 (仅用于仿真)
        sub_ = nh.subscribe("/ground_truth/state", 10, 
                           &FromTopicEstimate::callback, this);
    }
    
private:
    void callback(const nav_msgs::Odometry::ConstPtr& msg) {
        stateEstimate_.head(3) = toVector3d(msg->pose.pose.position);
        stateEstimate_.tail(3) = toVector3d(msg->twist.twist.linear);
    }
    
    ros::Subscriber sub_;
};
```

## 4. 参数调节

### 4.1 过程噪声 Q

**物理意义**: IMU 积分的不确定性

**调节方法**:
1. **位置噪声**: 0.01 - 0.1
   - 太小: 过于信任 IMU，漂移快
   - 太大: 过于依赖校正，震荡

2. **速度噪声**: 0.1 - 1.0
   - 太小: 速度估计不平滑
   - 太大: 速度响应慢

### 4.2 观测噪声 R

**物理意义**: 运动学校正的不确定性

**调节方法**:
1. **脚位置噪声**: 0.01 - 0.1
   - 太小: 过于信任运动学，对模型误差敏感
   - 太大: 校正效果弱，漂移快

2. **考虑因素**:
   - 机器人模型精度
   - 关节编码器精度
   - 地面平整度

### 4.3 初始协方差

**推荐值**:
```yaml
initial_covariance_position: 1.0   # 米²
initial_covariance_velocity: 1.0   # (米/秒)²
```

**说明**:
- 初始不确定性较大
- 滤波器会快速收敛

## 5. 性能优化

### 5.1 计算效率

**优化策略**:
1. **稀疏矩阵**: 利用 F, H 的稀疏性
2. **预计算**: 缓存不变量 (如 R)
3. **向量化**: 使用 Eigen 的向量化操作

### 5.2 数值稳定性

**技巧**:
1. **协方差正则化**:
```cpp
covariance_ += 1e-6 * Matrix6d::Identity();
```

2. **对称化**:
```cpp
covariance_ = 0.5 * (covariance_ + covariance_.transpose());
```

3. **正定性检查**:
```cpp
if (covariance_.eigenvalues().minCoeff() < 0) {
    covariance_ = Matrix6d::Identity();  // 重置
}
```

## 6. 常见问题

**Q: 估计位置漂移很快？**
A:
1. 增加过程噪声 Q
2. 减小观测噪声 R
3. 检查 IMU 校准
4. 检查接触检测

**Q: 估计值震荡？**
A:
1. 减小观测噪声 R
2. 增加过程噪声 Q
3. 检查 IMU 噪声
4. 检查运动学模型

**Q: 速度估计不准？**
A:
1. 调整速度过程噪声
2. 检查 IMU 加速度计
3. 增加低通滤波

**Q: 接触检测不准？**
A:
1. 调整接触力阈值
2. 使用力传感器
3. 融合多种检测方法

## 7. 高级主题

### 7.1 扩展卡尔曼滤波 (EKF)

**优势**: 处理非线性动力学
**劣势**: 计算量大，调参复杂

**适用场景**:
- 高速运动 (跑步、跳跃)
- 复杂地形
- 需要高精度

### 7.2 无迹卡尔曼滤波 (UKF)

**优势**: 不需要雅可比，精度高
**劣势**: 计算量大 (sigma 点)

**适用场景**:
- 强非线性系统
- 高精度要求

### 7.3 因子图优化

**优势**: 全局最优，处理延迟
**劣势**: 计算量大，实现复杂

**适用场景**:
- 离线处理
- 多传感器融合
- 长时间估计
