# 06. 与我们项目的对比分析

## 1. 项目对比概览

| 方面 | legged_control | 我们的项目 (embodied-ai-robot) |
|------|----------------|-------------------------------|
| **目标平台** | 四足机器人 (Unitree A1) | 人形机器人 (8 DOF 双腿) |
| **控制架构** | NMPC + WBC + 状态估计 | 简单 PD + IK |
| **动力学模型** | 质心动力学 + 完整动力学 | 简化模型 |
| **优化求解** | SQP + HPIPM | OSQP (未使用) |
| **状态估计** | 卡尔曼滤波 | 无 |
| **约束处理** | 摩擦锥、接触约束 | 无 |
| **Sim2Real** | 成熟的迁移流程 | 未实现 |
| **实时性** | NMPC 5ms, WBC 1ms | 未测试 |
| **代码成熟度** | 生产级，多实验室验证 | 原型级，仿真未通过 |

## 2. 控制架构对比

### 2.1 legged_control 架构

```
用户输入 (cmd_vel, goal)
    ↓
TargetTrajectoriesPublisher
    ↓
NMPC (100Hz)
  ├─ 预测时域: 1.5秒
  ├─ 优化变量: 质心动量 + 接触力
  ├─ 约束: 摩擦锥、接触、关节限位
  └─ 求解器: SQP + HPIPM
    ↓
State Estimation (500Hz)
  ├─ IMU 积分
  ├─ 运动学校正
  └─ 卡尔曼滤波
    ↓
WBC (500Hz)
  ├─ 决策变量: [q̈, f_c, τ]
  ├─ 任务层级: 7层
  └─ 求解器: HQP
    ↓
HybridJointInterface (1kHz)
  ├─ 前馈: WBC 扭矩
  └─ 反馈: PD 控制
    ↓
电机
```

### 2.2 我们的项目架构

```
用户输入 (cmd_vel)
    ↓
Gait Generator (100Hz)
  ├─ 步态生成
  ├─ ZMP 计算
  └─ COM 轨迹
    ↓
WBC (100Hz)
  ├─ IK 计算关节目标
  ├─ 简单阻抗控制
  └─ 无分层优化
    ↓
Joint Controller (100Hz)
  ├─ PID 控制
  └─ 阻抗控制
    ↓
Gazebo (1kHz)
```

### 2.3 关键差异

**差异 1: NMPC 层**
- **legged_control**: 有 NMPC，预测未来 1.5 秒，优化接触力
- **我们**: 无 NMPC，只有步态生成器，无法预测和优化

**差异 2: 状态估计**
- **legged_control**: 卡尔曼滤波，融合 IMU + 运动学
- **我们**: 无状态估计，基座位置/速度未知

**差异 3: WBC 实现**
- **legged_control**: 分层优化 (HQP)，7 层任务优先级
- **我们**: 简单阻抗控制，无分层优化

**差异 4: 约束处理**
- **legged_control**: 显式处理摩擦锥、接触约束
- **我们**: 无约束处理

## 3. 数学模型对比

### 3.1 动力学模型

**legged_control**:
```
完整动力学:
  M(q) * q̈ + C(q, q̇) * q̇ + g(q) = S * τ + J^T * f_c

质心动力学 (NMPC):
  ḣ_com = A * h_com + B * f_c + g
```

**我们**:
```
简化模型:
  τ = Kp * (q_des - q) + Kd * (0 - q̇) + gravity_compensation

其中 gravity_compensation 是错误的 (高估 13 倍)
```

### 3.2 优化问题

**legged_control**:
```
NMPC:
  min Σ (x - x_ref)^T Q (x - x_ref) + (u - u_ref)^T R (u - u_ref)
  s.t. 动力学约束、摩擦锥、接触约束

WBC:
  min Σ w_i * ||task_i||²
  s.t. 分层优先级
```

**我们**:
```
WBC (未使用):
  min τ^T * Q * τ
  s.t. M * q̈ + h = S * τ + J^T * f_c
  
实际使用:
  τ = Kp * (q_des - q) + Kd * (0 - q̇)
  (无优化)
```

## 4. 代码实现对比

### 4.1 主控制器

**legged_control** (`LeggedController.cpp`):
```cpp
void LeggedController::update(const ros::Time& time, const ros::Duration& period) {
    // 1. 状态估计
    updateStateEstimation(time, period);
    
    // 2. MPC 优化
    mpcMrtInterface_->updateObservation(currentObservation_);
    mpcMrtInterface_->evaluatePolicy(...);
    
    // 3. WBC 求解
    vector_t torque = wbc_->update(stateDesired, inputDesired, ...);
    
    // 4. 安全检查
    safetyChecker_->check(...);
    
    // 5. 发送命令
    for (size_t i = 0; i < numJoints_; ++i) {
        hybridJointHandles_[i].setCommand(torque[i], posDes[i], velDes[i]);
    }
}
```

**我们** (`wbc_node.cpp`):
```cpp
void WBCNode::controlLoop() {
    // 1. 构建状态
    wbc::RobotState state;
    state.q = joint_positions_;
    state.dq = joint_velocities_;
    
    // 2. 构建命令
    wbc::TaskCommand command;
    command.com_position_des = com_command_;
    
    // 3. 计算扭矩 (简单 PD)
    Eigen::VectorXd torques = wbc_->compute(state, command);
    
    // 4. 发布扭矩
    torque_publisher_->publish(torque_msg);
}
```

**差异**:
- legged_control 有状态估计、MPC、WBC、安全检查
- 我们只有简单 PD 控制

### 4.2 WBC 实现

**legged_control** (`WbcBase.cpp`):
```cpp
vector_t WbcBase::update(...) {
    // 构建 7 层任务
    Task floatingBaseTask = formulateFloatingBaseEomTask();
    Task noContactMotionTask = formulateNoContactMotionTask();
    Task frictionConeTask = formulateFrictionConeTask();
    Task baseAccelTask = formulateBaseAccelTask(...);
    Task swingLegTask = formulateSwingLegTask();
    Task contactForceTask = formulateContactForceTask(...);
    
    // 构建 HQP
    HoQp::Ptr hqp = std::make_shared<HoQp>(floatingBaseTask);
    hqp = std::make_shared<HoQp>(noContactMotionTask, hqp);
    hqp = std::make_shared<HoQp>(frictionConeTask, hqp);
    hqp = std::make_shared<HoQp>(baseAccelTask, hqp);
    hqp = std::make_shared<HoQp>(swingLegTask, hqp);
    hqp = std::make_shared<HoQp>(contactForceTask, hqp);
    
    // 求解
    return hqp->getSolutions();
}
```

**我们** (`whole_body_controller.cpp`):
```cpp
Eigen::VectorXd WholeBodyController::computeImpedance(...) {
    // 简单 PD 控制
    for (int i = 0; i < num_joints_; ++i) {
        double pos_error = q_des(i) - state.q(i);
        double vel_error = -state.dq(i);
        torques(i) = kp(i) * pos_error + kd(i) * vel_error;
    }
    return torques;
}
```

**差异**:
- legged_control 使用分层优化，处理 7 层任务
- 我们使用简单 PD，无优化

## 5. 性能对比

### 5.1 计算性能

**legged_control**:
- NMPC: 3-5ms (100-200Hz)
- WBC: 0.5-1ms (500Hz)
- 状态估计: <0.1ms (500Hz)
- 总延迟: ~10ms

**我们**:
- WBC: <1ms (100Hz)
- 关节控制: <1ms (100Hz)
- 总延迟: ~20ms

### 5.2 控制性能

**legged_control**:
- 可完成: 行走、跑步、跳跃、上下楼梯
- 鲁棒性: 可抵抗外部扰动
- 适应性: 可适应不同地形

**我们**:
- 可完成: 无 (仿真中无法站稳)
- 鲁棒性: 无
- 适应性: 无

## 6. 差距分析

### 6.1 核心差距

**差距 1: 缺少 NMPC 层**
- **影响**: 无法预测未来，无法优化接触力
- **后果**: 无法处理复杂运动，无法适应地形变化

**差距 2: 缺少状态估计**
- **影响**: 基座位置/速度未知
- **后果**: 无法进行精确控制

**差距 3: WBC 实现简单**
- **影响**: 无法处理多任务优先级
- **后果**: 无法同时满足动力学约束和跟踪任务

**差距 4: 无约束处理**
- **影响**: 无法保证摩擦锥、接触约束
- **后果**: 脚会滑动，接触力不合理

### 6.2 次要差距

**差距 5: 动力学模型不准确**
- **影响**: 重力补偿错误 (高估 13 倍)
- **后果**: 关节扭矩不合理

**差距 6: 无 Sim2Real 流程**
- **影响**: 无法迁移到真实机器人
- **后果**: 只能在仿真中运行

**差距 7: 参数调节不系统**
- **影响**: PD 增益随意设置
- **后果**: 性能不稳定
