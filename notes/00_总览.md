# 00. 整体架构概览

## 1. 系统框架

legged_control 采用**分层控制架构**，从上到下分为：

```
用户输入 (cmd_vel, goal, gait_command)
    ↓
TargetTrajectoriesPublisher (轨迹生成)
    ↓
NMPC (非线性模型预测控制) - 100Hz
    ↓
WBC (全身控制器) - 500Hz
    ↓
HybridJointInterface (关节控制) - 1kHz
    ↓
电机驱动
```

## 2. 核心模块

### 2.1 NMPC (Nonlinear Model Predictive Control)

**作用**: 预测未来状态，优化控制序列

**输入**:
- 当前状态 (基座姿态、关节角度、速度)
- 目标轨迹 (期望的质心轨迹、步态)

**输出**:
- 优化的状态轨迹 (未来 1.5 秒)
- 优化的接触力轨迹

**数学模型**:
$$
状态变量: x = [h_com, q_b, q_j]
  - h_com: 质心动量 (6维)
  - q_b: 基座位姿 (7维: 位置 + 四元数)
  - q_j: 关节角度 (12维)

控制输入: u = [f_c, v_j]
  - f_c: 接触力 (12维: 4脚 × 3维)
  - v_j: 关节速度 (12维)

优化问题:
  min Σ (x - x_ref)^T Q (x - x_ref) + (u - u_ref)^T R (u - u_ref)
  s.t. 动力学约束、摩擦锥约束、接触约束
$$

**求解器**: SQP (序列二次规划) + HPIPM

### 2.2 WBC (Whole-Body Controller)

**作用**: 处理全身动力学，分配关节扭矩

**决策变量**:
$$
x_wbc = [q̈, f_c, τ]
  - q̈: 广义坐标加速度 (18维)
  - f_c: 接触力 (12维)
  - τ: 关节扭矩 (12维)
$$

**任务层级** (从高到低):
1. **浮动基动力学** (严格满足): `M*q̈ + h = S*τ + J^T*f_c`
2. **无接触运动**: 支撑脚加速度 = 0
3. **摩擦锥约束**: 接触力在摩擦锥内
4. **基座加速度跟踪**: 跟踪 NMPC 的基座加速度
5. **摆动腿跟踪**: 跟踪参考脚轨迹
6. **接触力跟踪**: 跟踪 NMPC 的接触力
7. **关节扭矩最小化**: min `τ^T * W * τ`

**求解**: 分层二次规划 (HQP)

### 2.3 状态估计 (State Estimation)

**作用**: 融合 IMU + 运动学，估计基座状态

**估计状态**:
```
x_est = [p_b, v_b]
  - p_b: 基座位置 (3维)
  - v_b: 基座速度 (3维)
```

**传感器融合**:
- **IMU**: 基座姿态、角速度、线加速度
- **关节编码器**: 关节位置、速度
- **运动学**: 通过正运动学计算脚位置

**卡尔曼滤波**:
$$
预测步 (IMU 积分):
  p̂(k+1) = p̂(k) + v̂(k) * dt + 0.5 * R * a_imu * dt²
  v̂(k+1) = v̂(k) + R * a_imu * dt

更新步 (运动学校正):
  当脚触地时:
    z_foot_measured = FK(q)
    z_foot_predicted = p_b + R * r_foot
    校正: p_b = p_b + K * (z_foot_measured - z_foot_predicted)
$$

## 3. 数据流

```
┌──────────────┐
│  用户输入     │ cmd_vel, goal, gait_command
└──────┬───────┘
       ↓
┌─────────────────────────────────────────┐
│  TargetTrajectoriesPublisher            │
│  - 速度命令 → 质心轨迹                   │
│  - 生成步态相位                          │
└──────┬──────────────────────────────────┘
       ↓ referenceTrajectories
┌─────────────────────────────────────────┐
│  NMPC (OCS2)                            │
│  - 优化时域: 1.5秒, 50个节点             │
│  - 求解时间: ~5ms                        │
│  - 输出: 状态轨迹 + 接触力轨迹           │
└──────┬──────────────────────────────────┘
       ↓ stateDesired, inputDesired
┌─────────────────────────────────────────┐
│  StateEstimation (LKF)                  │
│  - 融合 IMU + 运动学                     │
│  - 估计基座位置、速度                    │
│  - 频率: 500Hz                           │
└──────┬──────────────────────────────────┘
       ↓ rbdStateMeasured
┌─────────────────────────────────────────┐
│  WBC (HQP)                              │
│  - 求解分层二次规划                      │
│  - 考虑完整动力学                        │
│  - 求解时间: ~1ms                        │
│  - 输出: 关节扭矩                        │
└──────┬──────────────────────────────────┘
       ↓ torque + pos_cmd + vel_cmd
┌─────────────────────────────────────────┐
│  HybridJointInterface                   │
│  - 前馈: torque (WBC)                    │
│  - 反馈: Kp*(q_des-q) + Kd*(dq_des-dq)  │
│  - 总扭矩: τ = τ_ff + τ_fb              │
└──────┬──────────────────────────────────┘
       ↓
┌─────────────────────────────────────────┐
│  电机驱动 (EtherCAT)                    │
│  - 频率: 1kHz                            │
└─────────────────────────────────────────┘
```

| NMPC | 100-200Hz | 3-5ms | NUC 11代 |
| WBC | 500Hz | 0.5-1ms | 任意现代CPU |

## 5. 关键设计决策

### 5.1 为什么用 NMPC + WBC 分层？

**NMPC 的优势**:
- 预测未来状态，提前规划
- 处理约束 (摩擦锥、关节限位)
- 优化能量消耗

**WBC 的优势**:
- 处理完整动力学
- 分层任务优先级
- 实时性好 (<1ms)

**分层的优势**:
- NMPC 处理长期规划 (1.5秒)
- WBC 处理短期跟踪 (当前时刻)
- 计算负担分散

### 5.2 为什么用质心动力学？

**质心动力学的优势**:
- 降维: 从 19维 (广义坐标) 降到 6维 (质心动量)
- 简化: 不考虑关节细节
- 实时: 求解速度快

**完整动力学的优势**:
- 精确: 考虑所有动力学效应
- 通用: 适用于各种运动

**结合使用**:
- NMPC 用质心动力学 (快速)
- WBC 用完整动力学 (精确)

### 5.3 为什么用分层优化 (HQP)?

**传统方法**: 加权最小二乘
$$
min Σ w_i * ||task_i||²
$$
**问题**: 权重难以调节，任务冲突时效果差

**分层优化**:
```
优先级 1: min ||task_1||²  (严格满足)
优先级 2: min ||task_2||²  (在 task_1 的零空间)
优先级 3: min ||task_3||²  (在 task_1, task_2 的零空间)
```
**优势**: 严格保证高优先级任务

## 8. 常见问题

**Q: NMPC 和 WBC 的区别？**
A: NMPC 预测未来 (1.5秒)，WBC 处理当前。NMPC 用简化模型 (质心动力学)，WBC 用完整模型。

**Q: 为什么需要状态估计？**
A: IMU 有漂移，需要运动学校正。卡尔曼滤波融合两者，得到准确的基座状态。

**Q: 为什么用分层优化？**
A: 保证高优先级任务 (如动力学约束) 严格满足，低优先级任务 (如扭矩最小化) 在零空间优化。

**Q: 如何调参？**
A: NMPC: 调整 Q, R 权重矩阵。WBC: 调整任务权重。状态估计: 调整卡尔曼增益。
