## 2. 核心文件详解

### 2.1 主控制器

**文件**: `legged_controllers/src/LeggedController.cpp`

**功能**: 整合所有模块，实现控制循环

**关键函数**:

```cpp
// 初始化
bool LeggedController::init(hardware_interface::RobotHW* robot_hw, 
                            ros::NodeHandle& controller_nh) {
    // 1. 获取硬件接口
    hybridJointHandles_ = ...;
    imuSensorHandle_ = ...;
    contactHandles_ = ...;
    
    // 2. 设置 NMPC 接口
    setupLeggedInterface(taskFile, urdfFile, referenceFile, verbose);
    
    // 3. 设置 MPC 求解器
    setupMpc();
    setupMrt();
    
    // 4. 设置状态估计
    setupStateEstimate(taskFile, verbose);
    
    // 5. 设置 WBC
    wbc_ = std::make_shared<WbcBase>(...);
    
    // 6. 启动 MPC 线程
    mpcThread_ = std::thread(&LeggedController::mpcThread, this);
    
    return true;
}

// 控制循环
void LeggedController::update(const ros::Time& time, 
                              const ros::Duration& period) {
    // 1. 状态估计
    updateStateEstimation(time, period);
    
    // 2. 获取 MPC 优化结果
    mpcMrtInterface_->updateObservation(currentObservation_);
    mpcMrtInterface_->evaluatePolicy(...);
    
    // 3. WBC 求解
    vector_t torque = wbc_->update(stateDesired, inputDesired, 
                                   rbdStateMeasured, mode, period);
    
    // 4. 安全检查
    safetyChecker_->check(...);
    
    // 5. 发送扭矩到电机
    for (size_t i = 0; i < numJoints_; ++i) {
        hybridJointHandles_[i].setCommand(torque[i], posDes[i], velDes[i]);
    }
}
```

**阅读重点**:
1. `init()` 函数: 理解模块初始化顺序
2. `update()` 函数: 理解控制循环
3. `updateStateEstimation()`: 理解状态估计流程

### 2.2 NMPC 接口

**文件**: `legged_interface/src/LeggedInterface.cpp`

**功能**: 定义最优控制问题 (OCP)

**关键函数**:

```cpp
// 设置最优控制问题
void LeggedInterface::setupOptimalControlProblem(...) {
    // 1. 设置模型
    setupModel(taskFile, urdfFile, referenceFile, verbose);
    
    // 2. 设置参考管理器
    setupReferenceManager(taskFile, urdfFile, referenceFile, verbose);
    
    // 3. 定义最优控制问题
    problemPtr_ = std::make_unique<OptimalControlProblem>();
    
    // 4. 添加代价函数
    auto cost = getBaseTrackingCost(taskFile, centroidalModelInfo_, verbose);
    problemPtr_->costPtr->add("baseTracking", std::move(cost));
    
    // 5. 添加约束
    auto frictionCone = getFrictionConeConstraint(...);
    problemPtr_->inequalityConstraintPtr->add("frictionCone", std::move(frictionCone));
    
    auto zeroVelocity = getZeroVelocityConstraint(...);
    problemPtr_->equalityConstraintPtr->add("zeroVelocity", std::move(zeroVelocity));
}

// 基座跟踪代价
std::unique_ptr<StateInputCost> LeggedInterface::getBaseTrackingCost(...) {
    // 状态权重
    matrix_t Q = matrix_t::Identity(stateDim, stateDim);
    Q.block<3, 3>(0, 0) *= weightBasePosition;
    Q.block<3, 3>(3, 3) *= weightBaseOrientation;
    Q.block<3, 3>(6, 6) *= weightBaseVelocity;
    
    // 输入权重
    matrix_t R = initializeInputCostWeight(taskFile, info);
    
    return std::make_unique<QuadraticStateInputCost>(Q, R);
}
```

**阅读重点**:
1. `setupOptimalControlProblem()`: 理解 OCP 构建
2. `getBaseTrackingCost()`: 理解代价函数设计
3. `getFrictionConeConstraint()`: 理解约束实现

### 2.3 WBC 实现

**文件**: `legged_wbc/src/WbcBase.cpp`

**功能**: 求解分层二次规划 (HQP)

**关键函数**:

```cpp
// WBC 更新
vector_t WbcBase::update(const vector_t& stateDesired,
                         const vector_t& inputDesired,
                         const vector_t& rbdStateMeasured,
                         size_t mode,
                         scalar_t period) {
    // 1. 更新测量状态
    updateMeasured(rbdStateMeasured);
    
    // 2. 更新期望状态
    updateDesired(stateDesired, inputDesired);
    
    // 3. 构建任务
    Task floatingBaseTask = formulateFloatingBaseEomTask();
    Task noContactMotionTask = formulateNoContactMotionTask();
    Task frictionConeTask = formulateFrictionConeTask();
    Task baseAccelTask = formulateBaseAccelTask(...);
    Task swingLegTask = formulateSwingLegTask();
    Task contactForceTask = formulateContactForceTask(...);
    
    // 4. 构建 HQP (分层)
    HoQp::Ptr hqp = std::make_shared<HoQp>(floatingBaseTask);
    hqp = std::make_shared<HoQp>(noContactMotionTask, hqp);
    hqp = std::make_shared<HoQp>(frictionConeTask, hqp);
    hqp = std::make_shared<HoQp>(baseAccelTask, hqp);
    hqp = std::make_shared<HoQp>(swingLegTask, hqp);
    hqp = std::make_shared<HoQp>(contactForceTask, hqp);
    
    // 5. 求解
    vector_t solution = hqp->getSolutions();
    
    // 6. 提取关节扭矩
    vector_t torque = solution.tail(numJoints_);
    
    return torque;
}

// 浮动基动力学任务
Task WbcBase::formulateFloatingBaseEomTask() {
    Task task(numDecisionVars_);
    
    // 动力学方程: M*q̈ + h = S*τ + J^T*f_c
    // 重写为: [M, -J^T, -S] * [q̈; f_c; τ] = -h
    
    matrix_t A(6, numDecisionVars_);
    vector_t b(6);
    
    A.block(0, 0, 6, 18) = pinocchioInterface_.getM().topRows(6);
    A.block(0, 18, 6, 12) = -pinocchioInterface_.getJ().transpose().topRows(6);
    A.block(0, 30, 6, 12) = -pinocchioInterface_.getS();
    
    b = -pinocchioInterface_.getH().head(6);
    
    task.A_ = A;
    task.b_ = b;
    task.d_ = VectorXd::Zero(6);  // 严格等式约束
    
    return task;
}
```

**阅读重点**:
1. `update()`: 理解 WBC 流程
2. `formulateFloatingBaseEomTask()`: 理解动力学约束
3. `formulateSwingLegTask()`: 理解摆动腿跟踪

### 2.4 状态估计

**文件**: `legged_estimation/src/LinearKalmanFilter.cpp`

**功能**: 融合 IMU + 运动学，估计基座状态

**关键函数**:

```cpp
// IMU 更新 (预测步)
void LinearKalmanFilter::updateFromImu(const ros::Time& time,
                                       const ros::Duration& period) {
    // 1. 获取 IMU 数据
    Vector3d a_imu = imuSensorHandle_.getLinearAcceleration();
    Matrix3d R = imuSensorHandle_.getOrientation();
    
    // 2. 转换到世界坐标系
    Vector3d a_world = R * a_imu - Vector3d(0, 0, 9.81);
    
    // 3. 预测步
    double dt = period.toSec();
    stateEstimate_.head(3) += stateEstimate_.tail(3) * dt 
                            + 0.5 * a_world * dt * dt;
    stateEstimate_.tail(3) += a_world * dt;
    
    // 4. 协方差预测
    Matrix6d F = Matrix6d::Identity();
    F.block<3, 3>(0, 3) = dt * Matrix3d::Identity();
    covariance_ = F * covariance_ * F.transpose() + Q;
}

// 接触更新 (更新步)
void LinearKalmanFilter::updateFromContact(const ros::Time& time) {
    for (size_t i = 0; i < 4; ++i) {
        if (contactHandles_[i].isContact()) {
            // 1. 计算脚的世界位置
            Vector3d p_foot_world = computeFootPosition(i);
            
            // 2. 残差
            Vector3d y = Vector3d::Zero() - p_foot_world;
            
            // 3. 卡尔曼增益
            Matrix3d H = Matrix3d::Identity();
            Matrix3d S = H * covariance_.block<3, 3>(0, 0) * H.transpose() + R;
            Matrix3d K = covariance_.block<3, 3>(0, 0) * H.transpose() * S.inverse();
            
            // 4. 状态更新
            stateEstimate_.head(3) += K * y;
            
            // 5. 协方差更新
            Matrix6d I_KH = Matrix6d::Identity();
            I_KH.block<3, 3>(0, 0) -= K * H;
            covariance_ = I_KH * covariance_;
        }
    }
}
```

**阅读重点**:
1. `updateFromImu()`: 理解预测步
2. `updateFromContact()`: 理解更新步
3. `computeFootPosition()`: 理解正运动学

## 6. 常见问题

**Q: 如何添加新的机器人模型？**
A:
1. 创建 URDF 文件
2. 修改 `legged_interface` 中的机器人参数
3. 调整代价权重

**Q: 如何调试 NMPC？**
A:
1. 检查优化是否收敛
2. 检查约束是否满足
3. 可视化优化轨迹

**Q: 如何调试 WBC？**
A:
1. 检查 HQP 求解是否成功
2. 检查任务优先级
3. 可视化接触力

**Q: 如何提高性能？**
A:
1. 减少预测时域
2. 减少 SQP 迭代次数
3. 使用多线程
